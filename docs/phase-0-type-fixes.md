## 0.1 — Add Missing Fields to `CompanySettings`

**File:** `types/business.ts` line 50–54

The SQL table `company_settings` has two columns the frontend type does not know about:
- `employer_tax_percentage` — payroll tax % added on top of salary
- `benefits_percentage` — benefits/insurance % added on top of salary

Both are displayed in the Organization page salary multiplier UI.

**Fix:**

```ts
// types/business.ts
export interface CompanySettings {
    overheadPercentage: number;
    bufferPercentage: number;
    yearlyFixedCost: number;
    employerTaxPercentage: number;   // ADD
    benefitsPercentage: number;      // ADD
}
```

Update `toCompanySettings()` in `lib/supabaseOrganization.ts`:

```ts
function toCompanySettings(row: Record<string, unknown>): CompanySettings {
    return {
        overheadPercentage: row.overhead_percentage as number,
        bufferPercentage: row.buffer_percentage as number,
        yearlyFixedCost: row.yearly_fixed_cost as number,
        employerTaxPercentage: row.employer_tax_percentage as number,   // ADD
        benefitsPercentage: row.benefits_percentage as number,          // ADD
    }
}
```

Update `upsertCompanySettings()` in `lib/supabaseOrganization.ts`:


## 0.2 — Add `capacityRole` to `Employee` Type; Remove Separate `Engineer` Type

**Files:** `types/business.ts`, `store/businessStore.ts`

Currently `Employee` and `Engineer` are separate types pointing to the same `employees` SQL table. `Engineer` has a `role: RoleType` (capacity pool bucket) that maps to `employees.capacity_role`. They need to be unified.

**Fix in `types/business.ts`:**

```ts
export interface Employee {
    id: string;
    name: string;
    role: string;                // job_role_id UUID as text (billing role)
    roleName?: string;           // ADD — denormalized display name (role_name column)
    capacityRole?: RoleType;     // ADD — capacity pool bucket (capacity_role column)
    monthlySalary: number;
    workableHours: number;
    costPerHour: number;         // READ-ONLY: GENERATED by DB — never write this
    status: 'Active' | 'On Leave' | 'Terminated';
}
```

The `Engineer` type and `INITIAL_ENGINEERS` mock data in `store/businessStore.ts` can be deprecated once capacity pool is sourced from employees with `capacityRole` set.

## 0.3 — Remove `cost_per_hour` from Supabase Insert/Update

**File:** `lib/supabaseOrganization.ts` lines 153–178

`cost_per_hour` is `GENERATED ALWAYS AS (monthly_salary / NULLIF(workable_hours, 0))`. Postgres will reject any attempt to insert or update it.

**Fix — remove from `insertEmployee`:**

```ts
export async function insertEmployee(e: Employee): Promise<void> {
    const { error } = await supabase.from('employees').insert({
        id: e.id,
        name: e.name,
        role: e.role,
        monthly_salary: e.monthlySalary,
        workable_hours: e.workableHours,
        // cost_per_hour — REMOVE: GENERATED column, DB computes it
        status: e.status,
    })
    if (error) throw new Error(error.message)
}
```

**Fix — remove from `updateEmployeeDB`:**

```ts
export async function updateEmployeeDB(e: Employee): Promise<void> {
    const { error } = await supabase
        .from('employees')
        .update({
            name: e.name,
            role: e.role,
            monthly_salary: e.monthlySalary,
            workable_hours: e.workableHours,
            // cost_per_hour — REMOVE: GENERATED column, DB computes it
            status: e.status,
        })
        .eq('id', e.id)
    if (error) throw new Error(error.message)
}
```

| *(missing)* | `notes` | Add `notes?: string` |

```ts
export interface Invoice {

    notes?: string;              // ADD
```

## 0.8 — Add `tenant_id` to Supabase Inserts (Organization Module)

**File:** `lib/supabaseOrganization.ts`

Currently none of the insert functions pass `tenant_id`. This works only because Supabase RLS is not yet enforced. Before enabling RLS, every insert needs the tenant ID.

Pattern for each insert function:

```ts
// Get tenant ID from store
import { useTenantStore } from '@/store/tenantStore'

// Inside each insert function, add:
const tenantId = useTenantStore.getState().activeTenantId
if (!tenantId) throw new Error('No active tenant')

// Add to every insert payload:
{ tenant_id: tenantId, ...otherFields }
```

Affected functions: `insertDepartment`, `insertRole`, `insertEmployee`, `insertGlobalOverhead`, `upsertCompanySettings`.

## 0.9 — Add `tenant_id` Filter to Supabase Selects

**File:** `lib/supabaseOrganization.ts` lines 78–84

Currently fetches all rows with no tenant filter. This will return data from all tenants once the DB has real multi-tenant data.

```ts
const tenantId = useTenantStore.getState().activeTenantId

// Add .eq('tenant_id', tenantId) to every select:
supabase.from('departments').select('*').eq('tenant_id', tenantId).order('created_at')
supabase.from('roles').select('*').eq('tenant_id', tenantId).order('created_at')
supabase.from('employees').select('*').eq('tenant_id', tenantId).order('created_at')
supabase.from('global_overheads').select('*').eq('tenant_id', tenantId).order('created_at')
supabase.from('company_settings').select('*').eq('tenant_id', tenantId).single()
```

> Also update the `company_settings` query — change `.eq('id', 'singleton')` to `.eq('tenant_id', tenantId)` once moving to multi-tenant mode.
